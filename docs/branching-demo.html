Base Branch Recommendations

📦

Feature Release

base: main

Release branches are cut from production to ensure a clean starting point.

🔥

Hotfix

base: main

Urgent fixes branch from production state for fastest path to deploy.

🛠

Ticket / Feature

base: release/*

Work branches off the current release branch and feed back into it.

⚙

Chore / Config

base: main

Maintenance and config changes branch from production for parity.

New Work Checklist

  • ✓

    Check for other Claude instances

    Ensure no other agent is actively working in the same repo or worktree.

  • ✓

    Worktree decision

    If another instance is active, create a worktree for isolated parallel work.

  • ✓

    Branch name (never invented)

    Use the exact branch name provided by the user — never invent or guess names.

  • ✓

    Base branch recommendation

    Recommend the correct base (main for releases/hotfixes, release/* for features).

  • ✓

    Version check (if release branch)

    Verify the version tag does not already exist to prevent collisions.

  • ✓

    Branch drift check

    Confirm main, staging, and development are aligned before starting new work.

  • ✓

    Pull latest

    Fetch and pull the latest state of the base branch before creating new work.

  • ■

    Create branch HUMAN GATE

    The branch is only created after the human confirms the name, base, and version are correct.
