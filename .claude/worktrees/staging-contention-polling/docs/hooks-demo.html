Hook System — 9 bash hooks that intercept dangerous git/gh commands before they execute

Pre-Tool Hooks — Ask / Deny / Allow

Hook Overview Each hook fires before its target command, checking branch targets and enforcing rules

check-git-commit.sh Ask
Intercepts: git commit
Flags commits targeting protected branches. Prompts for confirmation before any commit proceeds, ensuring you never accidentally commit to the wrong branch.
check-git-push.sh Deny / Ask
Intercepts: git push
Blocks direct pushes to production when a staging branch exists. Normal pushes to feature branches prompt for confirmation.
check-git-merge.sh Deny / Ask
Intercepts: git merge
Blocks merges into production branches. Also blocks merging non-production branches into development to keep the branch hierarchy clean.
check-gh-pr-create.sh Deny
Intercepts: gh pr create
Hard-blocks PRs targeting master, main, staging, or development. Only release branches are valid PR targets — no exceptions.
check-git-reset.sh Deny / Ask
Intercepts: git reset
Hard-blocks reset on production branches unless branch protection is explicitly unlocked. All other resets require confirmation.
check-gh-release.sh Ask
Intercepts: gh release create
Intercepts GitHub Release creation and prompts for human confirmation. Ensures releases are intentional and follow the deployment workflow.
check-gh-api-protection.sh Ask
Intercepts: gh api (branch protection)
Intercepts API calls that modify branch protection rules. Prevents accidental or unauthorized changes to protection settings.
check-git-branch-delete.sh Ask
Intercepts: git branch -d / -D
Intercepts branch deletion commands and requires confirmation. Guards against accidentally deleting important branches.
check-git-tag.sh Ask
Intercepts: git tag (staging/active)
Flags staging claim and clear operations. When tagging or deleting staging/active, warns about contention implications before proceeding.

Bypass Resistance Hooks defend against common patterns that try to slip past argument parsing

CWD Manipulation
Attempted bypass cd /tmp && git push origin main
# change directory to dodge repo detection
Hook catches it check-git-push.sh resolves the real repo root via git rev-parse, ignoring CWD tricks. DENIED.
Command Chaining (&&)
Attempted bypass echo "hello" && git push origin main
# hide push behind an innocuous command
Hook catches it Pre-tool hooks fire per-command, not per-shell-line. The git push is intercepted independently. DENIED.
Semicolon Chaining
Attempted bypass git push origin main; echo "done"
# append trailing command to confuse parsing
Hook catches it Hook parses the tool input arguments directly, not the raw shell string. Trailing commands are irrelevant. DENIED.
Multiple && Chains
Attempted bypass git fetch && git checkout main && git push origin main
# bury push deep in a chain
Hook catches it Each git command in the chain triggers its own hook check. The push is caught regardless of chain depth. DENIED.
Worktree Path Resolution
Attempted bypass git -C /path/to/worktree push origin main
# operate on a different worktree to dodge hooks
Hook catches it Hooks resolve the actual worktree root with git rev-parse --show-toplevel, catching pushes from any path context. DENIED.
